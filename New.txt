-New engine type Excel Template-driven document generation:
    Design and architect a flexible Excel Template Export System for an engineering/instrumentation web platform.

    The system must allow users to export ETL table data into arbitrary Excel structures/templates while keeping the user as free as possible to design the Excel layout manually.

    IMPORTANT DESIGN PRINCIPLES:
    - Excel owns presentation/layout only
    - The application owns data binding and document generation logic
    - Users should NOT need to write Excel macros/VBA
    - Users should NOT need programming knowledge
    - The export system must support highly variable engineering document structures
    - The export system must scale beyond simple column mapping

    ======================================================================
    CORE CONCEPT
    ======================================================================

    The architecture should separate:

    1. Excel Template
       - Pure XLSX layout/design
       - Formatting
       - Logos
       - Borders
       - Print settings
       - Formulas
       - Merged cells
       - Static labels/text

    2. Export Definition
       - Defines how ETL data binds into the template
       - Defines repeating areas
       - Defines grouping logic
       - Defines sheet creation logic
       - Defines pagination/overflow rules
       - Defines filtering/grouping
       - Defines dynamic region expansion

    3. ETL Data Sources
       - Data already exists as structured ETL tables
       - Export system only consumes those tables

    ======================================================================
    EXAMPLES THAT MUST BE SUPPORTED
    ======================================================================

    ----------------------------------------------------------------------
    EXAMPLE 1 — Instrument Datasheet
    ----------------------------------------------------------------------

    Structure:
    - One Excel sheet describes one instrument
    - Many fixed-position fields
    - Each field contains a single value
    - Example:
      - Tag
      - Service
      - Range
      - Unit
      - Vendor
      - Notes

    Source:
    - ETL table: Instrument_data_sheet

    Requirements:
    - Bind individual cells to ETL fields
    - Example:
      - Cell B4 -> Instrument_data_sheet.tag
      - Cell D8 -> Instrument_data_sheet.service
    - System generates one sheet/document per instrument
    - Must support fixed-position forms

    This should be modeled as:
    - Single Value Binding

    ----------------------------------------------------------------------
    EXAMPLE 2 — Instrument List
    ----------------------------------------------------------------------

    Structure:
    - Infinite/repeating table
    - Each row = one instrument
    - Each column = one field
    - Dynamic row expansion

    Source:
    - ETL table: Instrument_list

    Requirements:
    - User selects a template row
    - System repeats row dynamically for each record
    - Example:
      - A12 -> tag
      - B12 -> service
      - C12 -> unit
    - Template row becomes a repeating row template
    - Row insertion/expansion handled by export engine
    - Formatting must propagate correctly

    This should be modeled as:
    - Repeating Table Binding

    ----------------------------------------------------------------------
    EXAMPLE 3 — Assembly Type Sheets
    ----------------------------------------------------------------------

    Structure:
    - One sheet per assembly type
    - Sheet contains:
      1. Small materials table
      2. List of instruments using that assembly type

    Sources:
    - Assembly_type_standard
    - Instruments_Assembly_type

    Requirements:
    - Group data by assembly_type
    - Create one sheet per assembly type
    - Render repeating material table
    - Render repeating instrument list
    - Maximum number of instruments per sheet is limited by a defined Excel range
    - If exceeded:
      - Automatically create continuation sheet(s)

    Must support:
    - Grouping
    - Sheet-per-group generation
    - Multiple repeating regions in same sheet
    - Overflow/pagination rules

    ======================================================================
    UI/UX REQUIREMENTS
    ======================================================================

    The system should NOT behave like a simple CSV mapper.

    The UI should behave like a:
    - Template Binding Designer
    - Document Generation Designer

    The AI should design an interface with these concepts:

    ----------------------------------------------------------------------
    1. Excel Template Visual Preview
    ----------------------------------------------------------------------

    - Render uploaded Excel template visually
    - Allow clicking cells/regions
    - Show sheet structure
    - Allow selecting:
      - single cells
      - row templates
      - rectangular regions

    ----------------------------------------------------------------------
    2. Data Source Browser
    ----------------------------------------------------------------------

    Side panel showing ETL sources:

    Example:
    Instrument_list
     ├── tag
     ├── service
     ├── range
     ├── unit

    Users drag/drop or bind fields visually.

    ----------------------------------------------------------------------
    3. Binding Types
    ----------------------------------------------------------------------

    The interface must explicitly support multiple binding modes:

    A. Single Value Binding
    - Bind one field to one cell

    B. Repeating Table Binding
    - Bind row template to dataset
    - Repeat dynamically

    C. Grouped Section Binding
    - Create grouped sheets/documents
    - Multiple repeating regions per sheet

    D. Overflow/Pagination Rules
    - Create continuation sheets
    - Define maximum rows per region
    - Repeat headers automatically

    ----------------------------------------------------------------------
    4. Named Regions
    ----------------------------------------------------------------------

    The system should strongly consider named regions instead of relying only on coordinates.

    Examples:
    - MATERIAL_TABLE
    - INSTRUMENT_TABLE
    - HEADER_REGION

    Regions should carry behaviors:
    - repeating
    - grouped
    - paginated
    - fixed

    ======================================================================
    ARCHITECTURAL REQUIREMENTS
    ======================================================================

    The AI should propose:

    - Internal export definition schema
    - Data binding model
    - Runtime rendering engine
    - Template parsing architecture
    - Region abstraction system
    - Dynamic row insertion strategy
    - Pagination strategy
    - Sheet cloning strategy
    - Group-by rendering architecture
    - Validation/error handling
    - Mapping persistence
    - Future extensibility

    ======================================================================
    IMPORTANT CONSTRAINTS
    ======================================================================

    - Users must remain free to design arbitrary Excel layouts
    - Export engine must support engineering documents, not only flat reports
    - Must support:
      - merged cells
      - print layouts
      - formulas
      - logos/images
      - multi-sheet workbooks
      - continuation sheets
      - grouped documents
    - The system must NOT depend on VBA/macros
    - Logic belongs in the application, not inside Excel

    ======================================================================
    GOAL
    ======================================================================

    The final system should resemble a lightweight engineering-oriented document generation engine specialized for instrumentation and industrial documentation.

    The AI should focus heavily on:
    - abstraction design
    - UX
    - export architecture
    - scaling complexity
    - maintainability
    - future extensibility
    - avoiding tight coupling between templates and ETL sources
