# static/engine/js/i18n.js

**Description:** Lightweight EN/IT string table. Exposes `I18n = { t, setLang, getLang }`. Loaded globally via `base.html` before all other engine scripts.

## Index

| Lines | Symbol | Description |
|-------|--------|-------------|
| 3–163 | `I18n` IIFE | String table + language management |
| 4–83 | `_tables.it` | Italian string table |
| 84–162 | `_tables.en` | English string table |
| 165 | `_lang` | Current language, initialized from `localStorage` key `'im.lang'`, defaults to `'it'` |
| 167–169 | `t(key)` | Look up string; falls back to `'en'` then returns key if not found |
| 171–175 | `setLang(lang)` | Switch language and persist to `localStorage` |
| 177 | `getLang()` | Return current language code |

## Decisions

- **Defaults to Italian**: `localStorage.getItem('im.lang') || 'it'` — the primary user base is Italian engineers. Language is stored per-browser, not per-project.
- **Fallback chain `lang → en → key`**: guarantees the UI never shows `undefined`; the key itself is a readable fallback if a string is missing from both tables.
- **Loaded in base.html**: makes `I18n.t()` available on all pages (index + tool pages) so `AppShell` can render the settings modal anywhere.
- **No dynamic reload**: changing language requires a page reload or re-rendering components that use `t()`. The settings modal hint warns users about this.
